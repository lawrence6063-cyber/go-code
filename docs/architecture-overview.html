一、cogent 是什麼

一句話：cogent 是一個用 Go 語言從零實現的自主編碼 Agent 運行時——可理解為 Claude Code / Aider / Cursor Agent 的開源 Go 實現：一個活在終端裡、能讀寫程式碼、執行命令、多輪推理並自主完成編碼任務的 AI 助手。

它不只是「呼叫一次 LLM」，而是一套完整工程系統，涵蓋九大能力領域：

Agent 核心迴圈 串流 LLM 工具系統 多層安全 上下文工程 可觀測性 終端 UI MCP 擴展 評測

二、核心概念：ReAct 迴圈

整個系統的心臟是 internal/engine 驅動的 ReAct 迴圈（Reasoning + Acting）：「推理 → 呼叫工具 → 觀察結果 → 再推理」不斷循環，直到模型認為任務完成。

   使用者輸入
        │
        ▼
 ┌──────────────────────────────────────────────┐
 │            engine.Run （一輪任務）              │
 │                                                │
 │   ┌─→ ① 推理 Reasoning  ── 呼叫 LLM 串流       │
 │   │        │                                   │
 │   │        ▼                                   │
 │   │   模型輸出：文字增量 + 是否要用工具？        │
 │   │        │                                   │
 │   │        ├── 無工具 ──→ 完成，回覆使用者      │
 │   │        │                                   │
 │   │        ▼ 有工具                            │
 │   │   ② 行動 Acting    ── 權限閘門 → 執行工具   │
 │   │        │                                   │
 │   │        ▼                                   │
 │   └─ ③ 觀察 Observation ← 工具結果回流給模型    │
 │              （直到無工具呼叫 或 觸達 MaxSteps） │
 └──────────────────────────────────────────────┘

MaxSteps（預設 50）是防止無限迴圈的護欄。

四、執行內核（engine / agent / llm）

套件職責
engine執行內核，驅動 ReAct 迴圈。對外方法：Run（跑一輪對話）、Resume（從會話恢復續跑）、Undo（撤銷上一輪，含檔案還原）。透過 StreamEvent 事件流向上輸出。
agent封裝「單個自治 Agent 的一次完整運行」，供 engine 與 task 工具複用——子 Agent 派發的基礎。
llm對接 DeepSeek / OpenAI 相容的 Chat Completions 串流接口（SSE），Stream(ctx, req) → <-chan Delta。重試策略考究：只在建流階段重試，一旦開始吐 token 就不重試（避免破壞增量語義）。

事件流 StreamEvent（型別在 internal/types）把整個過程事件化：EventText（文字增量）、EventToolStart、EventToolResult、EventCompacted（上下文壓縮）、EventDone、EventError。

五、工具系統（tool）

Agent 透過工具與世界互動。8 個內建工具：

工具類型作用
read_file唯讀讀檔
list_dir唯讀列目錄
grep唯讀內容搜尋
find_files唯讀檔名搜尋
write_file寫入建立/覆寫檔案（產出 Diff）
edit_file寫入精確字串替換（產出 Diff）
bash執行執行 shell 命令（過沙箱）
task派發派發唯讀子 Agent 做探索（刻意不含自身以杜絕遞迴派發）

Tool 介面：Name / Description / InputSchema / Call / CheckPermission。Pool 負責註冊與去重（內建優先，防外部同名工具劫持）。Guard 裝飾器統一為「寫/執行類 + 全部 MCP 外部工具」注入權限閘門與 HITL。

六、安全縱深防禦（permission / sandbox / worktree）

本專案最見功力之處——多層設防、fail-closed：

模型想執行 bash「rm -rf build/」
    │
    ▼  ① permission：預設 ask → 觸發 HITL 中斷
    │     └─ 彈出 raw 選單：Approve / Always / Edit / Reject
    ▼  ② sandbox：ValidatePath 路徑必須在工作區內
    │     IsControlPlaneWrite 禁止寫 .cogent/ 控制面
    │     危險命令確定性攔截
    ▼  ③ worktree：高風險任務在 git worktree 隔離副本執行
    │     改動先進沙盒副本，diff 審閱後才合併
    ▼  執行
套件職責
permission權限模型與人類在環（HITL）中斷協議：Prompter / Interrupt / Resolution / Decision，動作 Approve / Edit / Reject。
sandbox工作區路徑校驗 + 命令執行安全邊界。
worktree用 git worktree 為高風險任務建立隔離工作副本。

無人值守場景（goal/loop）可設 COGENT_YES 自動批准，但危險命令仍被 sandbox 確定性攔截、worktree 隔離兜底——爆炸半徑可控。

七、上下文工程（session / memory / contextmgr / progress）

讓 Agent「記得住、接得上、不爆窗」：

套件職責落盤位置
session會話持久化：每輪對話與工具呼叫落盤為可恢復的 transcriptdata/
memory跨會話長期記憶：讀取專案約定與偏好，注入系統提示.cogent/memory
contextmgr上下文視窗計算與自動壓縮：接近視窗上限時把舊歷史摘要成狀態筆記再重注入，並有連續失敗熔斷—
progress跨 run 的待辦看板（Loop 的「倉庫記憶」）.cogent/progress.md

八、可觀測與成本（observe / cost）

  • observe：統一的 trace（追蹤）/ metric（指標）抽象與 exporter（file/stdout/otlp），預設零開銷 no-op。
  • cost：裝飾 observe.Meter，攔截 cogent.tokens 計數，按模型單價累計美元成本並分類累計 in/out token；驅動 loop 的 --max-cost 護欄，也供狀態列展示。

九、終端 UI 六項 + MCP 擴展

行級 TUI 六項能力（基於 raw 模式 + ANSI + 解耦狀態機）：

#能力關鍵套件
①串流富渲染（Markdown-lite + spinner + 工具分區）render.StreamMarkdown · streamview
②Diff 彩色渲染（工具產出 unified diff + 著色）render.ColorizeDiff · tool/diff.go
③權限確認菜單（raw 單選，文本回退）menu.go · prompter.go
④歷史 + Ctrl-R reverse-i-search（含持久化）history.Store · lineeditor.go
⑤@ / 斜杠命令補全（雙觸發符 + 命令路由）completion · slashcmd.go
⑥狀態列（模型 / token / 成本）statusbar.go · cost.go

MCP 擴展：mcp 套件連接並管理 MCP（Model Context Protocol）server，把外部工具適配成內建 Tool 介面注入工具池——外部工具一律過 Guard 權限閘門，視為不可信。

十、三種自主級別（關鍵設計）★

cogent 提供遞進的三種自主程度，這是它作為「Agent 運行時」的精髓：

  🧑 人在環           🤖 單目標自主          🚀 完全自治
  cogent run    ──→   cogent goal "…"   ──→   cogent loop
  交互式 REPL          給目標自主規劃執行         反覆從待辦看板取目標
  多輪對話             maker/reviewer 雙角色      直到清空或觸達預算上限
  resume 可續跑        實作者改碼 + 審查把關      --max-cost 成本護欄
命令自主級別說明
cogent run人在環互動交互式 REPL，多輪對話，每輪由使用者驅動；resume 可從中斷處恢復。
cogent goal "…"單目標自主給一個目標，自主規劃並執行到完成——採 maker/reviewer 雙角色迴圈。
cogent loop完全自治反覆從待辦看板 .cogent/progress.md 取目標執行，直到清空或觸達預算/次數上限。

十一、一次對話的完整資料流

使用者輸入 "幫我把 timeout 從 5 改成 30"
   │
   ▼ engine.Run
   │  組裝訊息 = 系統提示 + memory(專案約定) + 歷史 + 本次輸入
   ▼ llm.Stream ─────────→ EventText「我來修改 config.go」（串流渲染 Markdown）
   │  模型決定呼叫 edit_file
   ▼ Guard.CheckPermission → HITL 選單「Approve/Always/Edit/Reject」（raw TUI）
   │  使用者選 Approve
   ▼ sandbox.ValidatePath 校驗 → edit_file 執行 → 產出 unified diff
   ▼ EventToolResult（render.ColorizeDiff 綠加紅刪彩顯）
   │  結果回流給模型
   ▼ llm.Stream ─────────→ EventText「已完成，timeout 現為 30」
   ▼ EventDone
   │
   └─ 全程：session 落盤 · observe 記 trace+token · cost 累計美元
             · contextmgr 若接近視窗則 compact · 狀態列顯示 token/成本

十三、設計哲學與學習路徑

工程亮點

  1. 嚴格分層 + 依賴單向：types 是最內層葉子（零業務依賴），杜絕循環依賴。純邏輯（render/completion/history/contextmgr）全部可單測。
  2. 縱深防禦：permission（HITL）→ sandbox（路徑/命令）→ worktree（隔離）→ 唯讀子 Agent，層層設防，fail-closed。
  3. 串流優先：從 LLM 的 SSE 到事件流到終端渲染，端到端串流；重試只在建流階段，保護增量語義。
  4. 上下文工程：session（接得上）+ memory（記得住）+ contextmgr（不爆窗）三位一體。
  5. 可觀測 + 成本護欄：trace/metric 內建，--max-cost 讓自治迴圈花費可控。
  6. 遞進自主：run → goal → loop，同一內核三種放權。
  7. 可擴展：MCP 讓工具可插拔。

建議學習路徑

  1. 先跑起來：cogent run 感受互動 → cogent goal "…" 感受自主。
  2. 讀內核：internal/engine 的 ReAct 迴圈 → internal/llm 串流 → internal/tool 工具介面。
  3. 讀安全：Guard → permission → sandbox → worktree 的縱深防禦鏈。
  4. 讀上下文工程：session / memory / contextmgr。
  5. 讀終端 UI：配合 docs/repl-raw-and-at-completion-tutorial.html。
一句話總結：cogent 是一個麻雀雖小、五臟俱全的生產級自主編碼 Agent 運行時——把「LLM + 工具 + 安全 + 上下文 + 自治迴圈」這套現代 AI Agent 的完整範式，用乾淨的 Go 架構完整落地了。
