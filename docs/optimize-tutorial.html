1可靠性 · 任何单点抖动都不该升级为整任务失败

生产级 coding agent 长链路、强依赖外部 LLM API，瞬时故障是常态而非异常：429、网络抖动、5xx、工具实现 panic。核心命题是：把抖动「就地降级」，绝不让它逃逸成整进程崩溃。

R1 · LLM 建流重试退避 P0

原先一次 429 就让当前 ReAct 步进白白失败。现在在 llm 包内加一个可注入的 RetryPolicy：只对建流阶段的可重试错误（429 / 5xx / 网络超时 / 连接 reset）做指数退避 + jitter；ctx 取消与一般 4xx 一律不重试；流一旦开始吐 token 就不再重试（避免破坏增量语义）。

退避用 time.Timer+math/rand 标准库手写，sleep 内 select <-ctx.Done() 可被取消；默认 MaxAttempts=1（不重试），由 cmd 按 env 显式开启。

R2 / R3 · goroutine panic 兜底，单点 panic 不击穿进程 P0

这是无人值守长跑的最大可靠性缺口：内核多处启动 goroutine，任意一处 panic（畸形输入解引用、JSON 断言失败）都会击穿整个进程。改法是在每个 goroutine 边界加一道 recover 兜底，把 panic 降级为该任务/该工具的错误。

R2 · 内核 goroutine

safeGo(onPanic, fn) 包裹 engine.Run/Resume 与 llm.pump，panic → EventError + slog.Error，channel 正常关闭。

R3 · 并发批工具

safeRun 把单个工具（含 MCP 外部工具）panic 规整为配对完整的 ToolResult{IsError}，让模型当普通工具失败自我修正。

internal/engine/safego.go// recover 仅 goroutine 顶层兜底，不替代正常 error 流；
// 返回 interface{} 不假设其为 error、不命名为 err（遵循编码规范）。
func safeGo(onPanic func(v any), fn func()) {
	defer func() {
		if v := recover(); v != nil {
			onPanic(v)
		}
	}()
	fn()
}
边界铁律：recover 只用于 goroutine 顶层兜底，绝不用它替代正常 error 流（用户规范明确禁止 panic 作常规错误处理）；恢复后任务以失败收尾，而非「吞掉继续」。

R4 / R5 · 单次 LLM 超时 + resume 边界显式化 P1/P2

R4：streamAssistant 对每次 llm.Stream 包 context.WithTimeout（默认 120s），防「首 token 迟迟不来」长挂起，超时即本轮失败由 maxSteps 兜底。R5：把「半行 JSON 被跳过、后续完整行仍能恢复」这一隐含健壮性固化为测试——并诚实记录其边界：坏行须以换行自成一行（无换行的半行会被下次 append 拼接而双双损坏）。

2安全性 · 不依赖模型善意的确定性防线

S1 · ValidatePath 解析符号链接，堵 symlink 越界 P1

原来的路径校验只做 filepath.Clean + 字符串前缀。问题：工作区内若有一个指向区外的 symlink（link → /etc/passwd），字符串层面仍落在 root 内、通过校验，但实际 I/O 会穿透到区外。修法：在前缀校验之上对真实落点再 EvalSymlinks 一次。

目标不存在时解析其最近的已存在父目录（应对写新文件场景）；签名不变，所有文件工具透明受益。

S2 / S3 · 统一脱敏包，落盘与 trace 共用一套规则 P1

原来脱敏只覆盖 sk- + 几个 JSON 字段。新建 internal/secret 叶子包（仅标准库），把规则集中维护并扩充到 6 类凭据，供 session 落盘与 observe 入 trace 共用同一套规则——避免两处脱敏不一致。

sk-… OpenAI/DeepSeek
ghp_/gho_/ghu_/ghs_ GitHub PAT

AKIA… AWS Key
xox[baprs]- Slack

Bearer … / JWT eyJ…
-----BEGIN … PRIVATE KEY-----

S3 的关键约定：在 observe 包内对任何字符串型 span/metric 属性，入库前统一过 secret.Redact（集中拦截，避免每个埋点处遗漏）。这是 §3 补 span 字段工作的安全前置——先立脱敏约定，再补字段，否则补字段即引入泄露。

internal/observe/otel.gocase string:
	// 字符串值入 span/metric 前统一脱敏（S3 集中拦截）
	return attribute.String(a.Key, secret.RedactString(v))
依赖方向自洽：secret 仅依赖标准库，是最内层叶子包，被 session/observe 复用不构成循环依赖；脱敏拦截全部落在 observe 包内，守「只有 observe 碰 OTel」。

S4 / S5 / S6 · 危险命令扩充 + 取舍文档化 P2

S5 保守扩充危险命令清单（chmod -R 777 /、chown -R root /、base64 -d …|sh 编码绕过），并补测试断言不误拦合法的 rm -rf ./build。S4/S6 把「不做 OS 级强隔离的理由 + 残余风险」「Enabled=false 继承宿主环境的可信边界」写进 sandbox/doc.go，让安全姿态可审计、面试可解释。

3可观测性 · 追平自己许下的承诺

DEV_SPEC §8 早已画好一棵 span 树和一组指标蓝图。这一批的对标对象就是 cogent 自己——盘点代码追平了多少、补齐缺口。

O2 · Tracer 接口的「最小且向后兼容」扩展（先行）

要在 span 结束时补 token/ttft/outcome 这些「跑完才知道」的属性，但旧 EndFunc 只收 err。改法巧在用可变参数：

internal/observe/observe.go// 旧：type EndFunc func(err error)
// 新：可变参数 → 老调用点 end(err) / end(nil) 一行不改即兼容
type EndFunc func(err error, attrs ...Attr)
为什么低风险：变长参数为空时，所有现存 end(err)/end(nil) 调用点无需改动即可编译。只有需要补属性的点才传 attr。全仓 go build + go vet 一次通过，零测试替身改动。OTel 类型仍不外泄，no-op 仍零开销。

O1 / O3 / O4 / O5 · 补根 span、指标、稀疏属性、worktree span

借 O2 的能力，把 span 树与指标补齐到 §8.3/§8.7 承诺的水平：

新增指标：cogent.compact.count/circuit_open、cogent.llm.ttft、cogent.tool.duration、cogent.step.duration，全部 no-op 时零开销。
诚实边界：llm.finish_reason 当前流式 Delta 未透出，标注为暂未落地；total_steps/total_tokens/cost_usd 通过指标暴露而非 session span 属性。这些已在 DEV_SPEC §8.3/§8.7 的「承诺-落地对账」中如实记录（C2）。

5怎么开启与验证

新增能力默认fail-closed（最安全的一侧），由环境变量显式开启：

env# R1 LLM 建流重试（默认 1 = 不重试，向后兼容）
COGENT_LLM_RETRY_ATTEMPTS=3        # 最大尝试次数
COGENT_LLM_RETRY_BASE_MS=500       # 退避基准
COGENT_LLM_RETRY_MAX_MS=10000      # 退避上限

# R4 单次 LLM 调用超时（默认 120s）
COGENT_LLM_TIMEOUT_SEC=120
verify# 全量验证（每批结束都跑）
gofmt -l internal cmd      # 空输出 = 已格式化
go vet ./...               # 静态检查通过
go build ./...             # 全仓编译
go test -race ./...        # 19/19 包全绿 + goleak 无泄漏
go test -race -tags=integration ./internal/worktree/...   # 真 git 集成测试
git diff --stat go.mod go.sum   # 空 = 零新增依赖

6守住的五条不变量

不变量本轮如何守住
依赖只能向内新增 secret 为最内层叶子包；worktree span 在 cmd 层埋点，叶子包不反向依赖 observe；engine 仍不依赖 loop

7收官 · STAR 与诚实边界

S 情境
v1+v2 功能完整，但要「敢长期无人值守地跑」，可靠性/安全/可观测仍有缺口。

T 任务
以 Claude Code 为标杆，逐维度加固 23 项，零接口破坏、零新增依赖。

A 行动
退避重试 + panic 兜底 + symlink 防护 + 统一脱敏 + EndFunc 可变参数微扩 + 补 span/指标 + 文档对账。

R 结果
19/19 包 race 测试全绿、goleak 无泄漏、go.mod 零改动；OPTIMIZE_SPEC 升级 v1.0。

诚实边界（不粉饰）：
  • llm.finish_reason 未透出，span 暂缺该字段（已标注）；
  • session 级 total_steps/total_tokens/cost_usd 走指标而非 span 属性；
  • 沙箱仍是策略型隔离，未接 Linux landlock/seccomp 的 OS 级强隔离（已在 doc.go 显式声明残余风险）；
  • R5 的半行恢复仅对「以换行自成一行的坏行」成立，无换行的半行会被下次 append 拼接损坏。
一句话总结：这一轮没加任何功能，却把 cogent 从「能跑通 demo」推到了「敢被审计、敢被逐行追问、敢长期无人值守地跑」。详见 cogent/OPTIMIZE_SPEC.md §9 落地清单。
