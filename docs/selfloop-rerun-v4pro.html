3与上轮 reasoner 对比

同一套循环机制，仅换任务与模型，结局天差地别——直接印证「客观 verify 不可被主观短路」这一修复的价值。

上轮 · deepseek-reasoner

  • reviewer 输出大段思维链，首行非 APPROVED → 3/3 被判拒
  • reviewer 拒绝即短路 → goal.verify span 0 次，客观判据从未执行
  • maker 有 cwd 幻觉（cd /workspace），高延迟、高成本
  • 结局：budget_spent 未达成

本轮 · deepseek-v4-pro

  • reviewer 裁决格式合规 → approved=true
  • 客观 verify 每轮发射并执行 → verify.passed=true
  • maker 直接用相对路径编辑，1 轮给出正确实现
  • 结局：achieved，46.6s，$0.0384

注：即便 reviewer 这次主动批准，机制上 verify 仍是最终硬闸门—— 若 reviewer 误拒，只要 verify 通过依旧达标落盘（已由单测 TestRunGoal_ReviewerRejectButVerifyPasses 锁定）。

4单轮时间线

  • maker（v4-pro） 读取任务 → grep 定位 → edit_file 写入 switch 实现（n%15/%3/%5 + strconv.Itoa）
  • maker 自测 调 bash go test ./...
  • reviewer（v4-pro） 只读审查 → 裁决 APPROVED
  • 客观 verify（硬闸门） 在改动所在工作根跑 verify.sh → exit 0 · PASSED
  • 编排器 判 achieved，改动落盘，循环结束

maker 落盘的实现

func FizzBuzz(n int) string {
    switch {
    case n%15 == 0:
        return "FizzBuzz"
    case n%3 == 0:
        return "Fizz"
    case n%5 == 0:
        return "Buzz"
    default:
        return strconv.Itoa(n)
    }
}

5观察与结论

一处值得记录的小瑕疵（不影响结果）

maker 的收尾文本声称："因当前环境未安装 Go，无法自动运行测试……逐条对照用例全部一致"。 但事实上它已经调用了 bash go test ./... 工具。这是 v4-pro 残留的轻微 环境错觉——与上轮 reasoner 的 cwd 幻觉同源但更轻。 关键在于：这类自述完全不参与达标判定，最终以客观 verify.sh 退出码为准， 因此瑕疵被机制天然兜住。

结论

  • 修复后的客观 verify 硬闸门在真实运行中正确发射并主导达标判定（goal.verify 0→1）。
  • 裁决解析鲁棒化 + reviewer 选型让主观闸门不再恒拒；即便恒拒，客观判据也能独立闭合循环。
  • 成本护栏闭环产出真实可读的 cogent.loop.cost_usd（$0.0384），--max-cost 不再形同虚设。
  • finish_reason 端到端透出，实验目标 gap 已闭合。
  • v4-pro 在简单工程任务上遵循工具编排稳定、单轮收敛、成本与延迟显著优于 reasoner。

6复现方式

任务已恢复为初始 RED 态（fizzbuzz.go 为未实现桩），可直接复跑。

# 1) 基线：应为 FAIL（RED）
bash eval/tasks/implement_fizzbuzz/verify.sh

# 2) 构建 + 注入密钥（.env 不会被自动加载）
go build -o bin/cogent ./cmd/cogent
set -a; source .env; set +a

# 3) 用 v4-pro 跑自循环（带护栏与 trace）
cd eval/tasks/implement_fizzbuzz/repo
COGENT_MODEL=deepseek-v4-pro COGENT_REVIEWER_MODEL=deepseek-v4-pro \
COGENT_YES=1 COGENT_OBSERVE_ENABLED=true COGENT_TRACE_EXPORTER=file \
../../../../bin/cogent goal "$(cat ../task.txt)" \
  --review --verify "$(cd ..; pwd)/verify.sh" \
  --max-iterations 3 --max-cost 0.6 --max-wallclock 6m

# 4) 校验达标（GREEN）+ 观测指标
bash ../verify.sh
grep -o 'cogent.loop.outcome[^}]*' ../../../../data/traces/metrics-*.jsonl | tail -1
