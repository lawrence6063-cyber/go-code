为 cogent 新增一个只读内建工具 find_files，用于在工作目录内按 glob 模式递归查找文件名。

这是对 cogent 本体源码的改动。你需要自己探索代码库，理解现有内建工具的实现风格与注册方式，然后参照同样的模式新增 find_files 工具。

## 工具行为

- 工具名：find_files
- 入参：pattern（glob 模式，必填，对文件名 base name 匹配，使用 Go filepath.Match 语义）、
  path（检索起点目录，相对工作根，可选，为空表示工作根）
- 从 path 起递归遍历，收集所有"文件名匹配 pattern"的文件，
  输出为相对工作根的路径，每行一个，按字典序排序。
- 只读、并发安全。
- 越界路径（逃逸工作根）必须报错。
- 跳过噪声目录（.git、.cogent、node_modules 等）。
- 结果设上限保护，避免大仓库结果膨胀。
- 无匹配时返回友好提示，不报错。

## 验收标准（客观）

- go build ./... 通过
- go test ./... 全部通过
- find_files 在所有只读工具池装配点均完成注册（漏注册任何一处都会导致验收失败）
- find_files 能按 glob 递归命中文件、对越界路径报错、IsReadOnly 为 true

仅做上述相关改动；不要改动与本任务无关的代码，不要修改任何 verify 脚本或 eval/ 目录下的文件。
