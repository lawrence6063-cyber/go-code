## TUI 维度重构 + stream 输出优化（已完成 ✅）
用户诉求：最新 git `add tui` 引入的 TUI 能力太分散，要求「用 tui 维度聚合到一个文件夹」（最大化聚合，连 cost/SIGINT/REPL 编排一起搬）；并顺带修「stream 一股脑输出很乱」，要像 Claude Code：工具调用折叠单行摘要、正文/工具分区、隐藏入参 JSON 噪音、隔离子 Agent 事件。用户授权「可大刀阔斧重构，行为可变」。

### 落地结构（新 `internal/tui` 包，扁平同包 + 3 纯子包）
- 子包迁移：`internal/render|completion|history` → `internal/tui/{render,completion,history}`（git mv，纯标准库无互引；全局 import 前缀更新，引用点仅在原 cmd/cogent 内）。
- `cmd/cogent/lineeditor.go`(884行超限) 拆成 3 文件：`editor_core.go`(状态机/编辑/补全/历史)、`editor_render.go`(raw 读取/ANSI 重绘/下拉/CJK 宽字符)、`editor_keys.go`(decodeKey/rawInput/byteSource/errNotTerminal)。`lineeditor_raw.go`→`editor_raw.go`、`lineeditor_ioctl_{darwin,linux}.go`→`editor_ioctl_*.go`、`lineeditor_stub.go`→`editor_stub.go`（构建约束原样保留）。
- 迁移：`menu.go`、`prompter.go`、`cost.go`、`statusbar.go`、`slashcmd.go`（package main→tui）。
- 下沉编排：新增 `repl.go`（driveREPL/replLoop/inputLoop/runTurn/runTurnInterruptible/drainSignals/handleUndo + SIGINT 独占管理，从 commands.go 搬出）、`loopview.go`（RunLoopView/printLoopEvent/printInnerEvent/... 从 goal.go 搬出）、`tui.go`（包注释 + RunDeps/REPLOptions + RunREPL 入口）、`env.go`（envFloat 助手，原依赖 cmd）。
- 导出面：`InputReader`+`NewTTYInputReader`/`NewCLIPrompter`/`NewPrompter`；`CostProvider`+`NewCostProvider`+`CostMeter() loop.CostMeter`（cmd 不能访问私有 meter，用此访问器接 loop.Deps.Cost）；`StatusBar`+`NewStatusBar(cp CostProvider,model)`；`RunREPL`/`RunLoopView`/`Summarize`。其余同包私有符号不导出。
- 依赖方向：tui→engine(接口)/loop/permission/observe，均不反向依赖，无环。cliPrompter 仍实现 permission.Prompter；costMeter 仍实现 loop.CostMeter + observe 装饰链。

### cmd/cogent 变薄
commands.go/goal.go/loop.go/review.go 只保留 cobra 命令 + env 解析 + 运行时装配(buildEngine/ToolPool/MCP/Orchestrator)，交互/渲染改调 `tui.NewTTYInputReader`/`NewCLIPrompter`/`NewPrompter`/`NewCostProvider`/`NewStatusBar`/`RunREPL`/`RunLoopView`。review.go 的 `costProv.meter`→`costProv.CostMeter()`。

### stream 输出改造（核心 UX，重写 streamview.go→tui/stream.go）
- 新增 `toolsummary.go`：`Summarize(name, input json.RawMessage)` 按工具类型抽关键参数(read/write/edit/list→path、grep→pattern、bash→command、task→description)成单行，`render.Truncate` 截断，容错不 panic；**HITL 权限提示复用它**（不再打印完整 input JSON）。
- 渲染器维护 `inTool` 状态：EventToolStart→`● 工具名 摘要`(cyan)并置位、flush 正文+空行分区；工具区间内 EventText(进度/子 Agent 文本)**抑制**不并入正文；EventToolResult→单行状态 `⎿ ✓ +A -B / N lines / 单行内容截断 / ✗ 错误前3行`，复位 inTool。子 Agent(task) 汇聚在 ToolResult.Content 经强折叠只显规模。非 TTY(rich=false) 保留纯文本 `[tool]/[result]` 简明路径，可管道消费。
