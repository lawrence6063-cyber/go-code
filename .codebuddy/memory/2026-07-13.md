- **诊断**：我们 prompt-scaffold 净零，是因 v4-pro 单发已 60%、提示压不过采样噪声；命中率上限由单采样随机性决定，必须引入测试时计算。
- **方案分阶段（映射 cogent，不改内核）**：A 候选采样 best-of-N（加 `COGENT_TEMPERATURE` env 透传 llm.Request.Temperature，跑 N=5/temp0.7 收集 N 补丁）【最高 ROI 先做】→ D 去重+多数投票 rerank → B 复现测试选择（LLM 生成复现脚本，在 swebench 实例 Docker 内跑，留使其通过的候选）【核心信号】→ C 回归筛选（镜像内跑基线通过测试子集，丢破坏回归者）→ E 结构化定位(BM25/embedding+LLM 文件排序注入提示，可选)。里程碑：先 A+D 见效→测 Pass@N vs Pass@1 gap→上 B 吃掉 gap。
- **度量**：Resolved@1(best-of-N) 需多次重复压噪声才算有效；+ Pass@N / 复现选择准确率 / 补丁聚焦度 / 成本。
